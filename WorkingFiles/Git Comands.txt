****************************** Основные Git команды ******************************

* Все Git команды ты можешь выполнять через контекстное меню в Idea(щелкнуть пр. кнопкой мыши по файлу в проекте или папке) и выбрав пункт Git

1) Git Add - если ты добавляешь новый файл которого не было в проекте, git о нем ничего не знает(не следит за ним), чтобы этот стал отслеживаемым нужно выполнить Add, в Idea такие файлы подсвечены зеленым цветом, те файлы за которыми git не следит красные или серые

2) Git commit - если ты в существующем файле делаешь изменения(модифицируешь) он будет подсвечен синим цветом или зел. если только что добавлен, ты эти файлы можешь зафиксировать(закоммитеть), при commit ты указываешь кооментарий(что ты сделал с этими файлами) и они сохранаяются у тебя на машине, на сервер(github) они еще не попадают

3) Git Push - ты можешь сделать несколько коммитов, если решишь ты можешь залить их на сервер, перед заливкой git проверит, имеются ли на сервере изменения(сделанные к примеру мной), если изменений нет он зальет, если изменения есть он попросит сделать merge(слияние файлов), merge либо пройдет успешно, тогда ты просто еще раз повторяешь push, либо с конфликтом, тогда нужно руками править конфликт(к примеру ты меняешь какой-то файл, а я уже тоже его поменял и залил на сервер) следовательно может быть конфликт

4) Git Pull - получить информацию с сервера(gitHub), ровно также может быть merge

5) Git Revert - откатить изенения сделанные тобой до последнего успешного commit-а

6) Git diff - разница между файлами, текушими измененными и последними сохраненными

7) Git log - информация о всех комитах

Ты всегда можешь вернуться на пред. комиит
